# 网络设备自动巡检工具 - 日志系统更新说明

## 更新内容

已成功调整程序的日志输出路径，在 `\InspectionLogs\logs` 文件夹中输出debug信息和所有日志，同时保持巡检输出的文件不变。

## 新的日志系统特性

### 1. 多级别日志记录
- **DEBUG级别**: 详细的调试信息，用于问题排查
- **INFO级别**: 一般信息，程序运行状态
- **WARNING级别**: 警告信息
- **ERROR级别**: 错误信息
- **CRITICAL级别**: 严重错误信息

### 2. 日志文件输出位置

#### 系统日志文件（新增）
- **位置**: `\InspectionLogs\logs\`
- **all_logs_时间戳.log**: 包含所有级别的日志信息
- **debug_时间戳.log**: 仅包含DEBUG级别的详细调试信息
- **debug.log**: 兼容性文件，保持原有的debug日志格式

#### 巡检结果文件（保持不变）
- **位置**: `\InspectionLogs\YYYY_MM_DD\`
- **设备名_IP_时间戳.txt**: 每台设备的巡检结果
- **设备名_IP_session_时间戳.log**: 设备连接会话日志

### 3. 日志记录改进

#### 程序启动日志
- 记录程序启动时间和环境信息
- 记录Python版本和运行路径
- 记录日志系统初始化状态

#### 设备连接日志
- 使用INFO级别记录设备连接开始和完成
- 使用ERROR级别记录连接失败和错误信息
- 使用DEBUG级别记录详细的连接过程

#### 巡检过程日志
- 使用INFO级别记录巡检进度和结果
- 使用ERROR级别记录巡检错误
- 使用DEBUG级别记录命令执行详情

### 4. 便捷的日志记录函数

程序现在提供了便捷的日志记录函数：
```python
debug_log(message, level="DEBUG")  # 可指定级别的通用函数
log_info(message)                  # INFO级别日志
log_warning(message)               # WARNING级别日志  
log_error(message)                 # ERROR级别日志
```

### 5. 错误处理改进

- 程序启动失败时，错误日志会保存到 `\InspectionLogs\logs\inspection_error.log`
- 所有异常都会记录详细的堆栈信息到ERROR级别日志
- 网络连接异常会被分类记录（认证失败、连接超时、其他错误）

## 使用说明

### 查看日志
1. **实时日志**: 程序界面的日志窗口显示实时信息
2. **详细调试**: 查看 `\InspectionLogs\logs\debug_时间戳.log`
3. **完整日志**: 查看 `\InspectionLogs\logs\all_logs_时间戳.log`
4. **巡检结果**: 查看 `\InspectionLogs\YYYY_MM_DD\设备名_IP_时间戳.txt`

### 故障排查
1. 程序运行异常时，优先查看ERROR级别日志
2. 设备连接问题时，查看对应设备的session日志
3. 深度调试时，查看DEBUG级别日志的详细信息

## 向后兼容性

- 保持了原有的debug.log文件，确保现有的日志查看习惯不受影响
- 巡检结果文件的格式和位置完全不变
- UI界面的日志显示功能保持不变

## 技术实现

使用Python标准库的logging模块实现：
- 多处理器支持（文件和控制台输出）
- 自动日志轮转和时间戳
- 线程安全的日志记录
- 灵活的日志级别过滤

这个更新提供了更强大和专业的日志系统，便于程序维护和问题排查，同时保持了良好的向后兼容性。
