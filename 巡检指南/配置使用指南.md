# 网络设备自动巡检工具配置使用指南

## 一、配置文件概述

本工具使用三个主要配置文件：

1. **设备类型配置文件**：定义支持的设备类型及其连接参数
2. **设备列表配置文件**：列出需要巡检的具体设备
3. **命令配置文件**：定义每种设备类型需要执行的巡检命令

## 二、设备类型配置文件（device_types）

### 文件路径
`config/device_types.csv`（优先读取csv文件）
`config/device_types.txt`

### 格式说明
```
类型ID|名称|Netmiko SSH驱动|Netmiko Telnet驱动|是否需要enable(1/0)|禁用分页命令|默认协议|命令文件路径
```

### 字段解释
- **类型ID**：设备类型的唯一标识符
- **名称**：设备类型的显示名称
- **Netmiko SSH驱动**：使用SSH连接时的Netmiko驱动类型
- **Netmiko Telnet驱动**：使用Telnet连接时的Netmiko驱动类型
- **是否需要enable(1/0)**：设备是否需要进入特权模式（1=需要，0=不需要）
- **禁用分页命令**：用于禁用设备输出分页的命令
- **默认协议**：默认连接协议（ssh或telnet）
- **命令文件路径**：该设备类型对应的命令配置文件路径

### 配置示例
```
# 设备类型配置文件
# 格式: 类型ID|名称|Netmiko SSH驱动|Netmiko Telnet驱动|是否需要enable(1/0)|禁用分页命令|默认协议|命令文件路径
0|华为设备|huawei|huawei_telnet|0|screen-length 0 temporary|ssh|commands/commands_huawei.txt
1|思科设备|cisco_ios|cisco_ios_telnet|1|terminal length 0|ssh|commands/commands_cisco.txt
2|Juniper设备|juniper_junos|juniper_junos_telnet|0|set cli screen-length 0|ssh|commands/commands_juniper.txt
3|H3C设备|hp_comware|hp_comware_telnet|0|screen-length disable|ssh|commands/commands_h3c.txt
```

## 三、设备列表配置文件（devices）

### 文件路径
`config/devices.csv`（优先读取csv文件）
`config/devices.txt`

### 格式说明
```
设备名|IP|类型ID|用户名|密码|enable密码|端口|协议(可选)
```

### 字段解释
- **设备名**：设备的标识名称
- **IP**：设备的IP地址
- **类型ID**：对应设备类型配置文件中的类型ID
- **用户名**：登录设备的用户名
- **密码**：登录设备的密码
- **enable密码**：进入特权模式的密码（如果需要）
- **端口**：连接端口
- **协议(可选)**：连接协议（ssh或telnet），如果不指定则使用设备类型的默认协议

### 配置示例
```
# 设备列表文件
# 格式: 设备名|IP|类型ID|用户名|密码|enable密码|端口|协议(可选)
# SSH连接
思科交换机1|192.168.139.251|1|cisco|password|en_passwd|22|ssh

# Telnet连接
思科交换机2|192.168.139.251|1|cisco|password|en_passwd|23|telnet

# 使用默认协议(SSH)
华为防火墙1|192.168.139.254|0|op|password|22
```

## 四、命令配置文件（commands/*.txt）

### 文件路径
`config/commands/`目录下的各个命令文件

### 格式说明
- 每行一条设备巡检命令
- 以`#`开头的行将被忽略（注释行）
- 可以根据需要启用或禁用特定命令

### 配置示例（commands_huawei.txt）
```
# 华为设备巡检命令
# display version
# display interface brief
# display ip routing-table
# display arp all
display current-configuration
# display lldp neighbor-information
# display environment
```

### 配置示例（commands_cisco.txt）
```
# 思科设备巡检命令
# show version
# show interfaces
# show interfaces status
# show ip route
# show arp
show running-config
# show cdp neighbors detail
# show environment
```

## 五、使用步骤

1. **配置设备类型**：编辑`config/device_types.txt`文件，确保包含所有需要支持的设备类型
2. **配置设备列表**：编辑`config/devices.txt`文件，添加需要巡检的设备信息
3. **配置巡检命令**：根据需要编辑`config/commands/`目录下的命令文件
4. **启动工具**：运行程序，工具会自动加载默认配置文件
5. **选择设备**：在界面中勾选需要巡检的设备
6. **开始巡检**：点击"开始巡检"按钮执行巡检任务

## 六、注意事项

1. **编码问题**：工具支持多种编码格式，如遇到中文乱码问题，请检查设备的编码设置
2. **权限问题**：确保配置的用户名和密码具有足够的权限执行巡检命令
3. **网络连接**：确保工具运行环境能够访问配置的设备IP和端口
4. **防火墙设置**：如遇到连接问题，请检查防火墙是否阻止了相关端口的访问
5. **日志查看**：巡检结果保存在`InspectionLogs`目录下，可以查看日志了解巡检详情
