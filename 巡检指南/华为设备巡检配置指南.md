# 华为设备巡检配置指南

## 配置文件说明

### 1. 设备类型配置 (device_types.csv)

华为设备类型配置：
```csv
0,华为设备,huawei,huawei_telnet,0,screen-length 0 temporary,ssh,commands/commands_huawei.txt
```

**配置说明：**
- **类型ID**: 0
- **设备名称**: 华为设备
- **SSH驱动**: huawei (Netmiko华为SSH驱动)
- **Telnet驱动**: huawei_telnet
- **Enable模式**: 0 (华为设备通常不需要enable模式)
- **分页命令**: screen-length 0 temporary
- **默认协议**: SSH
- **命令文件**: commands/commands_huawei.txt

### 2. 命令配置文件 (commands_huawei.txt)

华为设备常用巡检命令包括：

#### 系统基本信息
- `display version` - 显示系统版本信息
- `display device` - 显示设备信息
- `display esn` - 显示设备序列号
- `display clock` - 显示系统时钟

#### 接口状态
- `display interface brief` - 显示接口简要信息
- `display interface description` - 显示接口描述
- `display interface` - 显示详细接口信息
- `display port vlan` - 显示端口VLAN信息

#### 路由和转发
- `display ip routing-table` - 显示路由表
- `display arp` - 显示ARP表
- `display mac-address` - 显示MAC地址表
- `display bgp peer` - 显示BGP邻居状态

#### 系统资源
- `display cpu-usage` - 显示CPU使用率
- `display memory-usage` - 显示内存使用情况
- `display temperature` - 显示温度信息
- `display power` - 显示电源状态

#### 配置和日志
- `display current-configuration` - 显示当前配置
- `display saved-configuration` - 显示保存的配置
- `display logbuffer` - 显示日志缓冲区
- `display alarm` - 显示告警信息

## 特殊配置说明

### 华为设备的特殊处理

#### 1. 连接参数
- **延迟因子**: 3（适应华为设备响应特性）
- **读取超时**: 90秒
- **连接超时**: 30秒
- **认证超时**: 30秒

#### 2. 命令执行策略
- 标准send_command方式
- 自动分页处理（screen-length 0 temporary）
- 支持华为特有的命令格式

#### 3. 认证机制
- 支持用户名密码认证
- 通常不需要enable模式
- 支持SSH密钥认证（部分型号）

## 使用注意事项

### 1. SSH/Telnet配置
```bash
# 华为设备SSH配置示例
ssh server enable
ssh user admin authentication-type password
ssh user admin service-type stelnet
```

### 2. 用户权限要求
- **基本查看权限**: 普通用户即可
- **配置查看**: 可能需要管理员权限
- **系统信息**: 建议使用管理员账号

### 3. 网络配置
```bash
# 确保管理接口配置正确
interface MEth0/0/0
 ip address 192.168.1.10 255.255.255.0
 undo shutdown
```

### 4. 性能考虑
- **执行时间**: 华为设备巡检通常需要3-8分钟
- **并发数建议**: 不超过10个并发连接
- **命令间隔**: 设备较忙时可能需要适当延迟

## 故障排查

### 1. 连接失败
- 检查SSH/Telnet服务：`display ssh server status`
- 验证网络连通性：`ping 管理IP`
- 检查用户配置：`display aaa local-user`

### 2. 认证失败
- 验证用户名密码
- 检查用户权限：`display aaa local-user admin`
- 查看认证日志：`display logbuffer | include login`

### 3. 命令执行超时
- 检查设备负载：`display cpu-usage`
- 检查内存使用：`display memory-usage`
- 适当增加超时时间

### 4. 权限不足
- 确认用户权限级别
- 使用具有足够权限的账号
- 检查AAA配置

## 安全建议

1. **用户管理**: 创建专用巡检账号，设置适当权限
2. **访问控制**: 使用ACL限制管理访问源
3. **协议安全**: 优先使用SSH而非Telnet
4. **日志审计**: 启用操作日志记录
5. **定期维护**: 定期检查和更新设备固件
