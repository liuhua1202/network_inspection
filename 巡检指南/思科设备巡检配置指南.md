# 思科设备巡检配置指南

## 配置文件说明

### 1. 设备类型配置 (device_types.csv)

思科设备类型配置：
```csv
1,思科设备,cisco_ios,cisco_ios_telnet,1,terminal length 0,ssh,commands/commands_cisco.txt
```

**配置说明：**
- **类型ID**: 1
- **设备名称**: 思科设备
- **SSH驱动**: cisco_ios (Netmiko思科IOS驱动)
- **Telnet驱动**: cisco_ios_telnet
- **Enable模式**: 1 (思科设备需要enable模式)
- **分页命令**: terminal length 0
- **默认协议**: SSH
- **命令文件**: commands/commands_cisco.txt

### 2. 命令配置文件 (commands_cisco.txt)

思科设备常用巡检命令包括：

#### 系统基本信息
- `show version` - 显示系统版本信息
- `show inventory` - 显示硬件清单
- `show clock` - 显示系统时钟
- `show running-config | include hostname` - 显示主机名

#### 接口状态
- `show interface status` - 显示接口状态
- `show interface description` - 显示接口描述
- `show interface` - 显示详细接口信息
- `show interface trunk` - 显示Trunk接口信息

#### 路由和交换
- `show ip route` - 显示路由表
- `show arp` - 显示ARP表
- `show mac address-table` - 显示MAC地址表
- `show cdp neighbors detail` - 显示CDP邻居详细信息

#### 系统资源
- `show processes cpu` - 显示CPU使用率
- `show memory` - 显示内存使用情况
- `show environment` - 显示环境状态
- `show power` - 显示电源状态

#### VLAN和生成树
- `show vlan brief` - 显示VLAN简要信息
- `show spanning-tree` - 显示生成树状态
- `show spanning-tree summary` - 显示生成树摘要

#### 路由协议
- `show ip ospf neighbor` - 显示OSPF邻居
- `show ip bgp summary` - 显示BGP摘要
- `show ip eigrp neighbors` - 显示EIGRP邻居

## 特殊配置说明

### 思科设备的特殊处理

根据记忆中的项目规范，思科设备享受以下优化：

#### 1. 连接参数优化
- **延迟因子**: 3（适应思科设备响应特性）
- **读取超时**: 90秒
- **连接超时**: 30秒
- **认证超时**: 30秒

#### 2. Enable模式处理
- **自动Enable**: 程序会自动进入enable模式
- **Secret密码**: 使用配置的secret密码进入特权模式
- **权限验证**: 确保在特权模式下执行命令

#### 3. 命令执行策略
- **三层重试机制**: 标准执行→提示符匹配→timing方式
- **分页处理**: 自动执行`terminal length 0`禁用分页
- **错误处理**: 智能处理命令执行错误

## 使用注意事项

### 1. SSH/Telnet配置
```bash
# 思科设备SSH配置示例
ip ssh version 2
ip ssh time-out 60
ip ssh authentication-retries 3
crypto key generate rsa modulus 1024
```

### 2. 用户权限配置
```bash
# 本地用户配置
username admin privilege 15 secret 0 cisco123
enable secret enable123

# AAA配置（可选）
aaa new-model
aaa authentication login default local
aaa authorization exec default local
```

### 3. 管理接口配置
```bash
# 管理VLAN配置
interface vlan 1
 ip address 192.168.1.20 255.255.255.0
 no shutdown

# 或管理接口配置
interface management 0
 ip address 192.168.1.20 255.255.255.0
 no shutdown
```

### 4. 性能考虑
- **执行时间**: 思科设备巡检通常需要3-8分钟
- **并发数建议**: 不超过8个并发连接
- **命令响应**: 复杂命令可能需要较长时间

## 故障排查

### 1. 连接失败
- 检查SSH/Telnet配置：`show ip ssh`
- 验证网络连通性：`ping 管理IP`
- 检查接口状态：`show interface vlan 1`

### 2. 认证失败
- 验证用户名密码
- 检查用户权限：`show running-config | include username`
- 检查enable密码：`show running-config | include enable`

### 3. Enable模式失败
- 验证enable secret密码
- 检查用户权限级别
- 确认AAA配置正确

### 4. 命令执行超时
- 检查设备负载：`show processes cpu`
- 检查内存使用：`show memory`
- 适当增加超时时间

## 安全建议

1. **访问控制**: 配置管理访问ACL
```bash
access-list 10 permit 192.168.100.0 0.0.0.255
line vty 0 15
 access-class 10 in
```

2. **用户管理**: 使用强密码和适当权限
```bash
username admin privilege 15 secret 0 ComplexPassword123!
```

3. **协议安全**: 禁用不必要的服务
```bash
no ip http server
no ip http secure-server
no cdp run  # 如果不需要CDP
```

4. **日志记录**: 启用详细日志
```bash
logging buffered 64000
logging console warnings
logging monitor warnings
```

5. **SNMP安全**: 使用SNMPv3或禁用SNMP
```bash
no snmp-server  # 如果不使用SNMP
```
