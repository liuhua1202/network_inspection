# 锐捷设备巡检配置指南

## 配置文件说明

### 1. 设备类型配置 (device_types.csv)

锐捷设备类型配置：
```csv
4,锐捷设备,ruijie_os,ruijie_os_telnet,1,terminal length 0,ssh,commands/commands_ruijie.txt
```

**配置说明：**
- **类型ID**: 4
- **设备名称**: 锐捷设备
- **SSH驱动**: ruijie_os (Netmiko锐捷驱动)
- **Telnet驱动**: ruijie_os_telnet
- **Enable模式**: 1 (锐捷设备需要enable模式)
- **分页命令**: terminal length 0
- **默认协议**: SSH
- **命令文件**: commands/commands_ruijie.txt

### 2. 命令配置文件 (commands_ruijie.txt)

锐捷设备常用巡检命令包括：

#### 系统基本信息
- `show version` - 显示系统版本信息
- `show clock` - 显示系统时钟
- `show inventory` - 显示硬件清单
- `show environment` - 显示环境信息

#### 接口状态
- `show interfaces` - 显示接口信息
- `show interfaces brief` - 显示接口简要信息
- `show interfaces description` - 显示接口描述
- `show interface status` - 显示接口状态

#### 路由和转发
- `show ip route` - 显示IP路由表
- `show arp` - 显示ARP表
- `show mac address-table` - 显示MAC地址表
- `show spanning-tree` - 显示生成树协议状态

#### 系统资源
- `show processes cpu` - 显示CPU使用率
- `show memory` - 显示内存使用情况
- `show power` - 显示电源状态
- `show temperature` - 显示温度信息

#### 配置和日志
- `show running-config` - 显示运行配置
- `show startup-config` - 显示启动配置
- `show logging` - 显示日志信息
- `show tech-support` - 显示技术支持信息

## 设备列表配置

### 基本格式
```
设备名|IP地址|设备类型ID|用户名|密码|secret|端口|协议
```

### 锐捷设备配置示例

#### 1. SSH连接（推荐）
```
锐捷交换机1|192.168.1.30|4|admin|admin123|enable123|22|ssh
锐捷路由器1|192.168.1.31|4|manager|pass123|enable456|22|ssh
```

#### 2. Telnet连接
```
锐捷核心交换机1|192.168.1.32|4|admin|admin123|enable123|23|telnet
```

#### 3. 非标准端口
```
锐捷防火墙1|192.168.1.33|4|admin|admin123|enable123|2222|ssh
```

**注意：锐捷设备通常需要enable密码，请确保在secret字段中填写正确的enable密码。**

## 特殊配置说明

### 锐捷设备的特殊处理

#### 1. 连接参数
- **延迟因子**: 2（锐捷设备响应适中）
- **读取超时**: 60秒
- **连接超时**: 30秒
- **认证超时**: 30秒

#### 2. Enable模式处理
- **自动进入enable模式**: 连接后自动执行enable命令
- **Enable密码**: 使用设备列表中的secret字段
- **提示符检测**: 支持用户模式和特权模式提示符

#### 3. 命令执行策略
- 标准send_command方式
- 自动分页处理（terminal length 0）
- 支持锐捷ROS系统命令格式

## 使用注意事项

### 1. SSH/Telnet配置
```bash
# 锐捷设备SSH配置示例
ip ssh server
ip ssh timeout 120
ip ssh authentication-retries 3
username admin privilege 15 password admin123
enable secret enable123
```

### 2. 用户权限要求
- **基本查看权限**: privilege 1-7
- **配置查看**: privilege 15
- **Enable模式**: 必须配置enable密码

### 3. 网络配置
```bash
# 确保管理接口配置正确
interface FastEthernet 0/1
 ip address 192.168.1.30 255.255.255.0
 no shutdown
```

### 4. 性能考虑
- **执行时间**: 锐捷设备巡检通常需要3-6分钟
- **并发数建议**: 不超过10个并发连接
- **命令间隔**: 部分型号可能需要适当延迟

## 故障排查

### 1. 连接失败
- 检查SSH服务：`show ip ssh`
- 验证网络连通性：`ping 管理IP`
- 检查用户配置：`show running-config | include username`

### 2. 认证失败
- 验证用户名密码
- 检查enable密码：`show running-config | include enable`
- 查看认证日志：`show logging | include login`

### 3. Enable模式失败
- 验证enable密码是否正确
- 检查用户权限级别
- 确认enable命令是否成功执行

### 4. 命令执行超时
- 检查设备负载：`show processes cpu`
- 检查内存使用：`show memory`
- 适当增加超时时间

## Enable模式相关

### 1. Enable模式检测
- 用户模式提示符：`设备名>`
- 特权模式提示符：`设备名#`

### 2. Enable命令执行
```bash
# 进入enable模式
enable
# 输入enable密码
[enable密码]
```

### 3. 常见Enable问题
- Enable密码错误
- 用户权限不足
- 网络延迟导致超时

## 安全建议

1. **用户管理**: 创建专用巡检账号，设置适当权限级别
2. **Enable密码**: 使用强密码保护enable模式
3. **访问控制**: 使用ACL限制管理访问源
4. **协议安全**: 优先使用SSH而非Telnet
5. **会话超时**: 设置合适的会话超时时间
6. **日志审计**: 启用操作日志记录
